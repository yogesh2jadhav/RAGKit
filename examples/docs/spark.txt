**Apache Spark** is an open-source distributed data processing engine designed to process very large datasets across multiple machines. It is widely used for **ETL, big data analytics, machine learning, graph processing, and streaming**.

At a high level, instead of processing data on a single computer, Spark divides the work into many small tasks and executes them in parallel across a cluster.

## How Spark works

```
             Spark Application
                    |
             +--------------+
             |    Driver    |
             +--------------+
                    |
          Schedules Tasks
                    |
        -----------------------
        |         |          |
   Executor 1  Executor 2  Executor 3
        |         |          |
     Partition  Partition  Partition
```

* **Driver**: Runs your application, creates the execution plan, and schedules tasks.
* **Executors**: Worker processes that execute tasks and store data in memory.
* **Cluster Manager**: Allocates resources (e.g., Standalone, Kubernetes, YARN).
* **Partitions**: Spark splits data into partitions so they can be processed in parallel.

---

## Why Spark is fast

Spark primarily keeps intermediate data **in memory** instead of repeatedly reading and writing to disk like traditional systems such as early Hadoop MapReduce.

For example:

Without Spark:

```
Read Disk
   ↓
Process
   ↓
Write Disk
   ↓
Read Again
   ↓
Process Again
```

With Spark:

```
Read Disk
   ↓
Load into Memory
   ↓
Process
   ↓
Process Again
   ↓
Write Result
```

This significantly reduces disk I/O for iterative workloads.

## Common transformations and actions

Transformations create a new dataset but don't execute immediately:

```python
df.filter(df.age > 30)
```

```python
df.select("name", "salary")
```

```python
df.groupBy("department")
```

Actions trigger execution:

```python
df.show()
```

```python
df.count()
```

```python
df.collect()
```

Spark uses **lazy evaluation**, meaning it builds a logical execution plan and only performs the work when an action is called.

---

## Fault tolerance

Spark tracks how datasets are derived (called **lineage**).

Example:

```
employees.csv
      |
   Filter
      |
 GroupBy
      |
 Aggregate
```

If a partition is lost because a worker fails, Spark recomputes only the missing partition using this lineage rather than rerunning the entire job.

# What Spark Really Is

Spark is **not a database**.

Spark is **not a storage system**.

Spark is a **distributed computation engine**.

Think of Spark as a giant distributed CPU.

```
             Storage
   (S3, ADLS, HDFS, Delta, Parquet)

                 ↓

           Apache Spark
     (Reads, Processes, Computes)

                 ↓

            Output Storage
```

Spark never owns your data permanently. It reads data from somewhere, processes it, and writes it somewhere.

# Adaptive Query Execution (AQE)

Spark 3.x introduced AQE.

At runtime Spark can

* change join strategy
* coalesce small partitions
* handle skewed data
* reduce shuffle

This is enabled by default in modern Databricks runtimes.

---

# Tungsten Engine

Spark's Tungsten engine improves execution by:

* Using efficient binary memory formats instead of Java objects where possible.
* Generating optimized bytecode for query execution (whole-stage code generation).
* Reducing garbage collection overhead.

These optimizations significantly improve CPU efficiency.

# Example: `groupBy`

```python
df.groupBy("department").count()
```

Execution flow:

```
Read Data
     │
     ▼
Split into Partitions
     │
     ▼
Each Executor Counts Local Departments
     │
     ▼
Shuffle Records by Department
     │
     ▼
Merge Counts
     │
     ▼
Return Result
```

The shuffle is the most expensive part because records with the same department from different executors must be brought together.
